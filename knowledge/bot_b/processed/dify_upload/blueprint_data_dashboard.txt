[2. Data Dashboard Introduction > 2.1 Final Solution Format]

Dmall Data Dashboard Stream is operated with SM guidance and Dmall technology. It aims to serve all the management and analytical data needs with product formats not limited to performance report, operation detail reports, big screen, app reports and other customized requests.

In comparison to operational reports, Performance reporting's audience is targeted to be management level and data analysts.

The application data service focuses on master data hierarchies including store hierarchy, merchandise hierarchy and vendor hierarchy level, rather than transaction level entries which are accessible in corresponding modules.

The data sources would be merging data from multiple domains for cross-sectional analysis. Trend analysis of historical data is supported with historical data migration project accomplished.

Functionality wise, Performance reports have rich presentation of descriptive statistics with comprehensively structured layout, auto aggregation and visualizations.

[2. Data Dashboard Introduction > 2.2 User Accessibility Matrix]

All reports that Dmall provides contain the whole dimensions and suitable measures according to different scenarios analysis (exclude some customized reports which the fixed format required). Compared with SM existing many reports, Dmall offers a clearer and simple solution by generating similar purpose within one chart to display on the system.

Consider the strict control over data access and acknowledging the high sensitivity of data permission requirements. Dmall designed the user access matrix function to match the SM policy. By utilizing this feature, Dmall’s reports enable administrators to configure various roles and, through role-based data permissions, systematically control the streamlined version of reports visible to different accounts in a batch manner.

Based on our current understanding of business, our data control extends support to individual stores, categories and movement types. Additionally, Dmall can implement strict restrictions on viewing for each column as needed.

[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities]

A standard report consists of time filter, dimension/attribute filter, reset button, LFL checkbox, column editor, download button, query button and data table section. The functions above are subject to be tailored for each standard report.

============================================================
[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities > 3.1.1.1 Time Filter]

Data Dashboard provides two types of time filters.

Absolute time range

In the Time Calendar, we can select range by Day / Week / Month / Year.

Relative time range (Fixed cannot define)

For corresponding TP time selection, the current day would be set as default, the same days last year is set as default in SPLY(same period last year).

============================================================
[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities > 3.1.1.2 Dimension/Attribute Filter]

For each report, there are multiple dimensions/attributes depending on the design of report. It may include master data dimensions as merchandise hierarchy, site hierarchy, vendor hierarchy and attributes such as article status, brand, sales type etc.

All the dimensions/attributes are enabled for data filtering in Data Dashboard standard report.  The data shown in corresponding table would be set by the dimension/attribute filter.

The filter is provided in two ways.

Dropdown list with search box and multiple selection enabled.

Text search box if option quantity is huge (now support max 50 options)

============================================================
[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities > 3.1.1.4 LFL checkbox]

LFL is the short form of like-for-like. When a store has transactions in all the last full 15 months, the store's status is considered as 'LFL' from the 1st of the next month.

LFL checkbox is basically a filter option for LFL stores. While the box is unchecked, there is no filtering from this checkbox. While the box is checked, it is equivalent to filter LFL stores' data only.

============================================================
[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities > 3.1.1.5 Column Editor / Summary Method]

Column editor button is on the top right corner of the data table.  It enables users to show/hide/change orders of some fields in the data table section.

Light blue shaded headers are dimension/attributes and aggregation supportable. The hidden dimension will be aggregated.

Headers with no shading are measures/dimension/attributes without aggregation. The hidden dimension will not be aggregated.

The column freeze button is supported.

============================================================
[3. SM Data Dashboard Solution > 3.1 Reports Design > 3.1.1 Common functionalities > 3.1.1.8 Drill Down Function]

In certain reports, highlighted data in blue offers a special interactive feature. Upon clicking, users can choose to drill down and analyze the data from different perspectives, such as transitioning from stores to products or from products to trend analysis. The drill-down feature may involve a pop-up window or a direct transition to another report, carrying the same filtering conditions for seamless analysis.

============================================================

[3. SM Data Dashboard Solution > 3.4 Addtional Function – Email Push & Print]

Dmall's advanced features include a robust system for email distribution and printing, offering administrators a streamlined approach to disseminating reports. Through this functionality, managers can effortlessly push reports to predefined email groups in a consistent format.

What sets this system apart is its intelligent backend mechanism. When reports are sent out, the system dynamically adjusts the content based on the data permissions of each recipient. This ensures that individuals only receive information that aligns with their specific access levels, maintaining data integrity, confidentiality, and relevance.

Furthermore, this multifaceted functionality extends to both email distribution and physical printouts, providing flexibility to accommodate diverse user preferences. The seamless integration of these features not only enhances the efficiency of report sharing within the organization but also upholds a high standard of data security and accessibility tailored to individual needs.
