# Dify 知识库操作指引

## 1. 访问 Dify

### 访问地址
```
http://localhost:3001
```

### 登录账号
- 首次登录时创建管理员账号
- 建议账号：`admin`
- 设置强密码并记录

### 登录后界面
```
顶部导航: 工作台 | 数据集 | 应用 | 探索 |
```

---

## 2. 创建知识库

### 步骤 1：进入数据集页面
点击顶部导航 **"数据集"**

### 步骤 2：创建知识库
点击右上角 **"创建知识库"**

### 步骤 3：选择导入方式
选择 **"上传文档"** 或 **"导入已处理的文档"**

推荐选择 **"导入已处理的文档"**，因为我们已预处理好了 JSON 文件。

### 步骤 4：填写知识库信息

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 名称 | `Bot A 故障处理知识库` | 知识库名称 |
| 描述 | `工单沉淀、PRD背景知识` | 可选 |

### 步骤 5：Embedding 配置

在 **"检索设置"** 中：

| 设置项 | 推荐值 | 说明 |
|--------|--------|------|
| 检索模式 | **混合检索** | 向量+关键词，效果最佳 |
| 相似度阈值 | 0.7 | 可调整 |
| Top K | 5 | 返回前5条 |

---

## 3. 导入文档

### 导入预处理好的文件

找到以下文件路径：
```
knowledge/bot_a/processed/
├── tickets.json   # 工单数据 (3326条)
├── prd.json      # PRD章节 (47个)
└── dict.json     # 数据字典 (32条)
```

**导入顺序：**
1. 先导入 `tickets.json`
2. 再导入 `prd.json`
3. 最后导入 `dict.json`

或者在 Dify 中合并为一个知识库导入。

### 导入步骤

1. 点击 **"导入文档"**
2. 选择 **"导入已处理的文档"**
3. 点击上传或拖拽 JSON 文件
4. 等待处理完成（根据数据量，可能需要几分钟）

### 查看导入进度

导入后可在知识库详情页查看：
- 文档数量
- 段落数量
- Embedding 状态

---

## 4. 配置知识库检索

### 混合检索设置（推荐）

在知识库的 **"检索设置"** 中：

```yaml
检索模式: 混合检索
  - 向量检索: 启用
  - 全文检索: 启用

权重分配:
  - 向量: 0.7
  - 关键词: 0.3

Top K: 5
相似度阈值: 0.7
```

### Rerank 重排序（可选）

如需更精准的结果，可启用 Rerank：
- 需要额外配置 Rerank 模型
- 可先不使用，后续按需开启

## 5. 在应用中使用知识库

### 创建 Bot A 应用

1. 进入 **"应用"** 页面
2. 点击 **"创建应用"**
3. 选择 **"聊天助手"**
4. 填写应用信息：
   - 名称：`Bot A - 故障处理`
   - 描述：`基于历史工单的知识库问答`

### 关联知识库

1. 在应用编辑页面，点击 **"添加功能"** 或 **"知识库"**
2. 选择已创建的知识库
3. 配置引用数量（建议 3-5 条）

### 配置 Prompt

示例 Prompt：
```
你是一个故障处理助手，基于知识库中的工单和PRD文档回答用户问题。

当知识库中有相关信息时，优先使用知识库内容回答。
如需补充背景知识，可适当引用PRD相关内容。

如果知识库中没有相关信息，请说明并建议用户联系技术支持。
```

## 6. 日常维护

### 添加新文档

1. 进入对应知识库
2. 点击 **"添加文档"**
3. 选择上传新文件
4. 等待处理完成

### 更新现有文档

1. 删除旧版本文档
2. 重新导入新版本
3. 确认知识库中无重复

### 检查知识库状态

定期检查：
- 文档数量是否正确
- Embedding 是否有失败
- 检索是否正常

---

## 7. 故障排查

### 文档导入失败

**可能原因：**
- 文件格式不对（需 JSON）
- 文件过大
- 网络问题

**解决方法：**
- 确认文件格式为 UTF-8 编码的 JSON
- 分批导入
- 检查 Dify 日志

### 检索无结果

**可能原因：**
- Embedding 未完成
- 相似度阈值过高
- 知识库为空

**解决方法：**
- 等待 Embedding 完成
- 降低相似度阈值
- 检查知识库是否有文档

### Dify 无法访问

**检查项：**
1. Docker 容器是否运行
2. 端口 3001 是否被占用
3. 执行 `init.sh status` 查看状态

## 8. 快速参考

| 操作 | 路径 |
|------|------|
| 创建知识库 | 数据集 → 创建知识库 |

```bash

./init.sh status

# 查看 Dify 日志
./init.sh logs api

# 重启服务
./init.sh restart
```
