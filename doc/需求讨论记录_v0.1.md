## 一、Bot 场景设计（已确认）

### 1.1 Bot 划分

| Bot | 定位 | 面向用户 | 知识来源 |
|-----|------|----------|----------|
| **Bot A - 故障处理** | 历史工单沉淀，快速查找解决方案 | Dmall Helpdesk + SM IT | 工单、问题记录（静态知识库） |
| **Bot B - 操作指南** | 系统蓝图、手册，正确使用指导 | 全体用户 | 用户手册、蓝图（静态知识库） |
| **Bot C - 版本指南** | 版本发布信息查询 | **仅 HQ IT + Dmall Helpdesk** | **MCP 飞书多维表格（实时查询）** |

### 1.2 Bot C 特别说明

- **不开放给门店用户**，仅总部 IT 和 Dmall Helpdesk 使用
- 用户群体为专业用户，问题更深度，假设有技术背景
- 通过 MCP 实时查询飞书多维表格，**不储存向量库**

## 二、知识库边界（已确认）

### 2.1 静态知识库（向量库）

| 内容类型 | 处理方式 | Chunk 策略 |
|----------|----------|------------|
| 工单 Q&A | 每条记录切分 300-500 字 | 重叠 50 字 |
| 手册/蓝图段落 | 每段落切分 500-800 字 | 重叠 100 字 |
| 版本发布记录 | **不进入静态知识库** | — |

### 2.2 MCP 飞书（动态查询）

- **用途**：Bot C 版本指南专属
- **数据源**：飞书多维表格（版本发布记录）
- **特点**：实时性强，按版本号或关键词查询，不做静态存储

### 2.3 飞书多维表格结构（待 SM 确认）

```
每条记录 = 一个版本发布
字段可能包含：
- 版本号（如 V3.5）
- 发布时间
- 审批记录
- 发布内容（多维字段）
```

**需要 SM 提供**：
1. 多维表格的 API Token 或 App Token
2. 表格字段结构文档

## 三、技术选型（已确认）

### 3.1 Embedding 模型

| 场景 | 推荐模型 | 说明 |
|------|----------|------|
| **中英混合为主** | BAAI/bge-m3 或 text2vec-base-multilingual | 工单和蓝图文档是英文，蓝图手册英文，需多语言支持 |

> ⚠️ 中文场景如果效果不好，可切换 text2vec-base-chinese

### 3.2 检索策略

**两级检索方案**：

```
用户问题
    ↓
[Level 1] 语义检索（向量相似度 Top 10-20）
    ↓
[Level 2] Re-rank（BAAI/bge-reranker 重新排序）
    ↓
[Level 3] 阈值过滤（相似度 < 0.5 丢弃）
    ↓
传给 LLM 生成答案
```

| 策略 | 说明 |
|------|------|
| 语义检索 | 向量相似度召回 |
| Re-rank | 提升相关性排序 |
| 阈值过滤 | 避免低质量检索结果进入生成 |

> ⚠️ Dify 原生支持向量检索和阈值设置；Re-rank 需额外配置或定制

### 3.3 Dify + 飞书对接方案

#### 飞书 API 调用路径

```
飞书多维表格
    ↓
飞书开放平台 API（获取 tenant_access_token）
    ↓
你的 API 函数（封装查询逻辑：按版本号查/按关键词查）
    ↓
Dify 自定义工具（HTTP 请求）
    ↓
Dify Agent（判断是版本问题 → 调用工具 → 整合结果 → 生成答案）
```

#### 实施步骤

1. **飞书侧**：
   - 在飞书开放平台创建应用
   - 获取 app_id + app_secret
   - 开通多维表格权限
   - 获取多维表格的 app_token 和 table_id

2. **Dify 侧**：
   - 在"工具"模块创建"自定义工具"
   - 配置 HTTP 请求（调用你的飞书 API 服务）
   - 定义工具描述和参数（让 Agent 知道什么时候调用）

3. **开发工作**：
   - 方案 A（推荐）：写一个轻量 API 服务封装飞书调用，Dify 通过 HTTP 调用
   - 方案 B：开发 MCP Server，Dify 通过 MCP 协议连接

## 四、答案溯源机制（已确认）

### 4.1 Prompt 设计

LLM 生成答案时要求显式标注来源：

```
你是一个知识库问答助手。请基于以下检索到的知识回答用户问题。

检索结果：
[1] 来源：SOP-SAP-001，片段："SAP上传失败处理流程..."
[2] 来源：WO-2024-0521，片段："问题原因：文件编码格式错误..."

要求：
1. 回答用户问题
2. 每引用一个检索结果，在句子后面加 [来源ID]
3. 不要引用不在上述检索结果中的来源
4. 如果检索结果不足以回答，诚实说明"根据现有知识无法回答"
```

### 4.2 来源卡片展示

```
答案：SAP上传失败通常由以下原因导致...

[来源1] SOP-SAP-001 - SAP上传标准流程
  "SAP上传失败处理流程：首先检查网络连接..."
  [查看详情]

[来源2] WO-2024-0521 - 工单记录
  "文件上传失败，解决方法：重新导出UTF-8格式"
  [查看详情]
```

### 4.3 用户反馈闭环

答案下方加反馈区域：

```
┌─────────────────────────────────────────────────────┐
│ 这答案有用吗？                                       │
│ 👍 有用    👎 没用                                   │
│                                                     │
│ （如果点"没用"，展开选项）                           │
│ ❌ 不相关  ❌ 来源错误  ❌ 答案不完整  ❌ 其他          │
│ 可选输入：_________________________                  │
└─────────────────────────────────────────────────────┘
```

## 五、用户反馈数据设计

### 5.1 反馈数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `question_id` | string | 问题ID |
| `question_text` | string | 用户原始问题 |
| `bot_id` | string | 哪个Bot |
| `answer_id` | string | 返回的答案ID |
| `rating` | enum | `useful` / `not_useful` |
| `reason` | enum | `not_relevant` / `wrong_source` / `incomplete` / `other` |
| `user_comment` | text | 用户补充说明（可选） |
| `user_id` | string | 用户ID（匿名或脱敏） |
| `timestamp` | datetime | 反馈时间 |
| `session_id` | string | 对话会话ID |

### 5.2 后台审核机制

```
反馈数据 → 入库（状态=pending）→ 审核员Review → resolved/declined
```

**审核功能**：
- 查看原始问答
- 标记问题类型（检索问题/生成问题/来源问题）
- 修正来源
- 触发知识库补充流程

### 5.3 数据应用

| 场景 | 说明 |
|------|------|
| 检索优化 | 某问题频繁被反馈不相关，调整阈值或优化chunk |
| 来源优化 | 某来源被标记错误，检查或替换 |
| 知识库补充 | 发现新问题没覆盖，触发补充流程 |
| 每周报告 | 汇总反馈数据，作为优化依据 |

## 六、错误处理与降级策略（已确认）

### 6.1 分层降级架构

```
用户提问
    │
    ▼
[Bot选择] ──→ 检查知识库是否可用
    │
    ├─── 可用 ──→ 检索 → 生成答案 → 返回
    │
    └─── 不可用
            │
            ▼
        [MCP 是否相关?]
            │
            ├─── 是 → 调用 MCP → 返回
            │         │
            │         └─── 失败 → 返回友好错误提示
            │
            └─── 否 → 返回静态兜底答案
                        │
                        └─── 无兜底 → "无法回答，建议联系IT"
```

### 6.2 各层错误处理

| 层级 | 错误场景 | 处理方式 |
|------|----------|----------|
| **LLM 响应** | 超时（>30s） | "服务繁忙，请稍后再试" |
| **LLM 响应** | 相似度低于阈值 | "未找到相关信息，建议联系IT支持" |
| **检索层** | Qdrant 不可用 | 返回静态兜底答案或提示服务异常 |
| **MCP 层** | 飞书 API 失败 | Bot C："暂时无法查询版本信息" |
| **MCP 层** | 飞书 API 超时 | "飞书服务响应慢，请稍后再试" |
